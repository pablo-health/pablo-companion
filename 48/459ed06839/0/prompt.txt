# Accessibility Foundation — Design Doc, Skill, and CLAUDE.md Rules

## Context
Pablo Companion has zero accessibility support today. Rather than just patching 70+ controls, we're building the foundation so **all future code is accessible by default**: a design doc (standards), CLAUDE.md rules (enforcement during coding), and an `/a11y` skill (audit existing code). We also address the Phase 3 color decisions.

---

## Deliverable 1: `docs/accessibility.md` — Accessibility Design Doc

Comprehensive standards doc covering:

### Section 1: Principles
- "Warm and welcoming for everyone" — accessibility doesn't mean bland
- Never rely on color alone to convey meaning
- Every interactive element must be usable via VoiceOver
- Respect system accessibility preferences (reduce motion, high contrast)

### Section 2: Required Modifiers by Control Type
| Control | Required | Example |
|---------|----------|---------|
| Button | `.accessibilityLabel()` | `"Start session with Jane Doe"` |
| Button (state-dependent) | `.accessibilityLabel()` that updates | `isPlaying ? "Stop" : "Play"` |
| Decorative Image | `.accessibilityHidden(true)` | PabloBear mascot, status icons |
| Informative Image | `.accessibilityLabel()` | Video platform icon → "Zoom call" |
| Custom control | `.accessibilityElement(children: .ignore)` + label + value | LevelMeter |
| Toggle | SwiftUI provides label automatically; add `.accessibilityHint()` if non-obvious |
| Picker | SwiftUI provides label; no extra work needed |
| Status badge | `.accessibilityElement(children: .combine)` | Groups icon + text |
| Animation | Check `@Environment(\.accessibilityReduceMotion)` | Pulsing avatar |
| Error message | `.accessibilityAddTraits(.isStaticText)` + announce via `AccessibilityNotification` |

### Section 3: Color Standards

**Default palette adjustment** — darken Honey slightly to pass WCAG AA:
- Honey: #E8A849 → **#D4922E** (~5.0:1 on white, passes AA, still distinctly warm amber)
- All other colors keep current hex values (already pass AA)

**High-contrast palette** (activated by manual toggle OR system "Increase Contrast"):
| Color | Default | High Contrast | Contrast on White |
|-------|---------|--------------|-------------------|
| Honey | #D4922E | #B87A1A | ~6.2:1 |
| Sage | #7A9E7E | #4A7A52 | ~5.8:1 |
| Sky | #89B4C8 | #4A7A9E | ~5.5:1 |
| Soft Brown | #6B5344 | #3D2E24 | ~12:1 |
| Error | #C45B4A | #A33A2A | ~7.2:1 |
| Blush | #E8B4A2 | #C47A6A | ~5.0:1 |
| Cream | #FDF6EC | #FDF6EC | (background, unchanged) |
| Deep Brown | #2C1810 | #2C1810 | (text, already 19.5:1) |

The high-contrast palette stays warm — deeper amber, richer forest green, warmer slate — not clinical grays. Still Pablo, just bolder.

**Implementation**: `Color+Pablo.swift` gets a static `highContrast` flag (from `@AppStorage` or environment). Each color property checks it:
```swift
static var pabloHoney: Color {
    highContrastEnabled ? Color(hex: "#B87A1A") : Color(hex: "#D4922E")
}
```

**System auto-detect**: In `ContentView`, observe `@Environment(\.accessibilityHighContrastEnabled)` and propagate to the color system.

### Section 4: Shape & Icon Differentiation (beyond color)
- "Start Session" button gets a play icon prefix
- "End Session" button gets a stop icon prefix
- Status badges already have text labels (good)
- Recording state: pulsing ring + "Recording" text indicator (not just green color)

### Section 5: Dark Mode
Explicitly out of scope for now. Document as future work. Keep `.preferredColorScheme(.light)` until a full dark-mode color mapping is designed.

## Deliverable 2: `/a11y` Claude Skill

A slash command that audits accessibility in Pablo Companion views.

**File**: `.claude/skills/a11y.md`

**What it does when invoked**:
1. Scans all `.swift` files in `mac/PabloCompanion/Views/`
2. Flags buttons without `.accessibilityLabel()`
3. Flags `Image()` without `.accessibilityHidden(true)` or `.accessibilityLabel()`
4. Flags custom controls without `.accessibilityElement()`
5. Flags animations without `accessibilityReduceMotion` checks
6. Reports findings grouped by severity (critical / warning / info)
7. Suggests specific fixes referencing `docs/accessibility.md` standards

**Skill file structure**: Prompt-based skill that uses Grep + Read to scan files and compare against the rules in the design doc.

## Deliverable 3: CLAUDE.md Additions

Add to the Swift / Xcode Conventions section:

```markdown
## Accessibility (non-negotiable)
- Every `Button` must have `.accessibilityLabel()` — use context like patient name
- Every decorative `Image` must have `.accessibilityHidden(true)`
- Custom controls: `.accessibilityElement(children: .ignore)` + `.accessibilityLabel()` + `.accessibilityValue()`
- Animations must check `@Environment(\.accessibilityReduceMotion)` before animating
- `.help()` is for tooltips, NOT accessibility — always pair with `.accessibilityLabel()`
- Never use color alone to convey meaning — always pair with text, icon, or shape
- See `docs/accessibility.md` for full standards and high-contrast palette
```
