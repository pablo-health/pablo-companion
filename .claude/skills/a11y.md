# Accessibility Audit — Pablo Companion

You are auditing the Pablo Companion SwiftUI views for accessibility compliance. Follow the standards defined in `docs/accessibility.md`.

## Steps

1. **Read the standards**: Read `docs/accessibility.md` to load the current accessibility rules.

2. **Scan all view files**: Use Grep to search all `.swift` files under `mac/PabloCompanion/Views/` for the following patterns:

### Critical Issues (must fix)

- **Buttons without accessibility labels**: Find `Button(` or `Button {` that are NOT followed (within ~5 lines) by `.accessibilityLabel(`. Every Button must have an explicit `.accessibilityLabel()`.

- **Images without accessibility handling**: Find `Image(` or `Image(systemName:` that are NOT followed by `.accessibilityHidden(true)` or `.accessibilityLabel(`. Every Image must either be hidden from VoiceOver (decorative) or have a label (informative).

- **Custom controls without accessibility**: Find custom view types like `StatusIndicator`, `LevelMeter`, or any `struct` conforming to `View` that contains interactive elements but lacks `.accessibilityElement(`.

### Warnings

- **Animations without reduce-motion check**: Find `.animation(`, `.withAnimation`, or `.transition(` that are NOT paired with `accessibilityReduceMotion` within the same view.

- **Color-only state indication**: Find patterns where color is the sole differentiator for state (e.g., `.foregroundStyle(isActive ? .green : .red)` without accompanying text or icon changes).

- **Generic or missing labels**: Find `.accessibilityLabel("")` or `.accessibilityLabel("Button")` or similar non-descriptive labels.

### Info

- **`.help()` without `.accessibilityLabel()`**: The `.help()` modifier is for tooltips, NOT for VoiceOver. Flag any control that has `.help()` but no `.accessibilityLabel()`.

- **High-contrast color usage**: Verify that `Color.pabloHoney`, `Color.pabloSage`, etc. are used (not hardcoded hex values), so the high-contrast palette is automatically applied.

3. **Read flagged files**: For each file with issues, read the relevant sections to confirm the finding and draft a specific fix.

4. **Report findings**: Group by severity and present as:

```
## Accessibility Audit Results

### Critical (N issues)
- **file.swift:NN** — Button "X" missing `.accessibilityLabel()`. Fix: add `.accessibilityLabel("Start session with patient name")`
- ...

### Warning (N issues)
- **file.swift:NN** — Animation without reduce-motion check. Fix: guard with `@Environment(\.accessibilityReduceMotion)`
- ...

### Info (N issues)
- ...

### Passing
- Files with no issues: file1.swift, file2.swift, ...
```

5. **Do NOT make changes** unless explicitly asked. This skill is audit-only by default.
