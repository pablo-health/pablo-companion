# Plan: Session Bugs, Transcription Resilience, and Channel Swap

## Context

Three bugs and three feature requests affecting the therapist experience:

**Bugs:**
1. Sessions always show "Unknown Patient" — backend returns `patient: null`, no client-side fallback
2. Session history filtering (Scheduled, In Progress, etc.) does nothing — backend likely ignores `status` query param
3. Quick start: after stopping recording, session still shows "Start Session" — `endSession()` never called, session stays at `.scheduled`

**Features:**
4. Mic/speaker swap toggle — for setups where mic captures patient and system audio carries therapist (default OFF)
5. Deferred transcription — if Whisper model not downloaded, queue sessions and process after download
6. Auto-retry + transcript viewing — resilient upload retry, surface pending status to therapist, wire existing transcript viewer

## 2. Bug Fix: Client-side session filtering

**Problem:** Backend may ignore `status` query param. All sessions returned regardless of filter.

**File:** `mac/PabloCompanion/ViewModels/SessionViewModel.swift` (lines 170-216)

**Changes:**
- After receiving `response.data` in `loadSessions()`, if `statusFilter` is non-nil, filter client-side:
  ```swift
  private func matchesFilter(_ session: Session) -> Bool {
      guard let filter = statusFilter else { return true }
      switch filter {
      case "scheduled": return session.status == .scheduled
      case "in_progress": return session.status == .inProgress
      case "recording_complete": return session.status == .recordingComplete
      case "finalized": return session.status == .finalized
      case "cancelled": return session.status == .cancelled
      default: return true
      }
  }
  ```
- Apply same filter in `loadMoreSessions()` when appending

**Changes:**

## 6. Feature: Auto-retry + transcript viewing

**Problem:** Upload retry only runs on app launch. Transcript viewer exists but isn't wired. Therapists need visibility into what's pending so they don't open support tickets.

**Files:**
- `mac/PabloCompanion/Views/ContentView.swift`
- `mac/PabloCompanion/Views/DayView.swift`
- `mac/PabloCompanion/ViewModels/TranscriptionViewModel.swift`

**Changes:**

### Wire recording completion → transcription
In `ContentView.configureAndLoad()`:
```swift
recordingVM.onRecordingCompleted = { [transcriptionVM] recording in
    transcriptionVM.transcribeIfNeeded(recording)
}
```

### Periodic auto-retry (every 5 minutes)
Add a `.task` on `authenticatedContent`:
```swift
.task {
    while !Task.isCancelled {
        try? await Task.sleep(for: .seconds(300))
        guard !Task.isCancelled else { break }
        await transcriptionVM.retryPendingUploads()
    }
}
```

### Exponential backoff for pending uploads
In `TranscriptionViewModel.retryPendingUploads()`, skip items where `retryCount` exceeds a threshold based on time since creation (e.g., retry at 5m, 15m, 1h, 4h). Simple approach: skip if `retryCount > 10` and flag for manual retry.

### Wire transcript viewer
Add to `authenticatedContent`:
```swift
.sheet(item: $viewingTranscript) { item in
    TranscriptViewerView(transcript: item.text, recordingDate: item.recordingDate)
}
```

### Status banners in Today tab + Sessions tab
Pass transcription state to both DayView and SessionHistoryView (counts + callbacks). Show non-intrusive banners:

- "2 transcripts pending upload" (with "Retry Now" button)
- "3 sessions awaiting model download" (with "Go to Settings" link)
- "Transcription failed" (with "Retry" button)

Banners stack below the recording banner (DayView) or below the filter bar (SessionHistoryView), only when relevant. Color-coded: honey for pending, terracotta for failed.

### Auto-retry with backoff + manual retry
- Auto-retry every 5 minutes via `.task` loop
- Exponential backoff: skip items where `retryCount` exceeds threshold (retry at 5m, 15m, 1h, 4h intervals). After 10 retries, stop auto-retrying and show "Retry" button only
- Manual "Retry Now" button in banner forces immediate retry of all pending items
- Goal: zero support tickets — the therapist either sees it resolve automatically or has a clear action to take

### Add "View Transcript" to session row
When a transcript is available (keyed by recording ID), show a small "View" button in `SessionRowView`. This requires mapping session ID ↔ recording ID — for now, pass a `transcriptText: String?` to `SessionRowView` and show button when non-nil.
